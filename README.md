# Courseware Note Skill

一个用于将课程课件 PDF 转换为标准化 Obsidian Markdown 学习笔记的 Codex Skill / Prompt 工作流。

本项目适用于需要长期、批量、稳定处理课程课件的场景。它的目标不是简单摘要，而是将课件内容完整重构为便于学习、复习、检索和长期保存的课程笔记。

## 项目定位

很多课程 PDF 课件本身更适合课堂展示，而不适合直接作为复习资料：

- 信息被拆散在多页 PPT 中；
- 同类知识点重复出现；
- 标题层级不稳定；
- 大量内容以短句、图示、表格或项目清单形式存在；
- 直接复制容易形成碎片化笔记；
- 过度总结又容易遗漏知识点。

本项目提供一套标准化处理规则，用于指导 AI 工具将课件 PDF 转换为结构统一、内容完整、阅读体验更好的 Markdown 笔记。

## 核心目标

将课程课件 PDF 转换为适合 Obsidian 使用的 Markdown 学习笔记。

处理结果应满足：

- 保留原课件中的全部实质知识点；
- 不改变原意；
- 不随意加入课件外内容；
- 去除课件式冗余；
- 避免 PPT 式碎片化表达；
- 避免将密集信息压缩成难读的长段落；
- 在段落、列表、表格、加粗小标题之间取得平衡；
- 生成适合学习、复习、检索和长期保存的 Markdown 笔记；
- 每份笔记末尾生成带 Obsidian 内部链接的学习自测。

## 适用场景

本项目适合：

- 课程课件 PDF 整理；
- 企业培训课件整理；
- 电力、工程、计算机、管理类课程笔记整理；
- Obsidian 知识库建设；
- 批量 PDF 转 Markdown 学习资料；
- 将课堂展示型材料转化为复习型笔记；
- 构建个人长期学习资料库。

不适合：

- 只想生成极短摘要；
- 需要完全逐字转录课件；
- 希望生成思维导图或 Mermaid 图；
- 希望 AI 大量扩写课件外知识；
- 需要保留原 PDF 视觉版式的场景。

## 项目特点

### 1. 不是摘要，而是结构化重写

本工作流不会将课件压缩成几段总结，而是尽可能保留全部实质知识点，并将其重构为更适合学习的笔记结构。

### 2. 不机械复制 PPT

PPT 中常见的短句、重复标题、分散页面和演示性文字会被重新组织，避免形成松散、割裂的 Markdown。

### 3. 保持内容完整

定义、分类、流程、参数、设备、案例、工程项目、操作要求、安全要求等实质内容都应保留。

### 4. 平衡段落、列表和表格

本项目特别强调：  
不要因为避免碎片化就完全不用分点，也不要为了整齐把所有内容都做成列表。

一般原则是：

- 段落用于解释原理、背景、意义和逻辑关系；
- 列表用于承载多个并列知识点、对象、措施、案例或参数项；
- 有序列表用于流程、步骤和操作顺序；
- 表格用于分类、对比和字段稳定的信息；
- 加粗小标题用于独立但不宜升级为正式标题的内容块。

### 5. 适配 Obsidian

输出 Markdown 会尽量保持标题稳定、链接清晰，并在学习自测中使用 Obsidian 内部链接关联正文知识点。

示例：

```markdown
关联原文：[[课程文件名#对应标题]]
````

### 6. 自动生成学习自测

每份笔记末尾会生成学习自测，题型包括：

* 单选题；
* 多选题；
* 判断题；
* 简答题。

每道题包含答案、解析和关联原文链接。

## 使用方式一：作为 Codex Skill 使用

将 `SKILL.md` 放入你的 Codex Skills 目录中。

示例目录：

```text
courseware-note/
└── SKILL.md
```

之后在 Codex 中调用：

```text
请使用 courseware-note skill，处理指定目录中的所有 PDF，并输出同名 Markdown 笔记。
```

也可以指定输入输出路径：

```text
请使用 courseware-note skill，处理 `D:\课程课件\待处理PDF` 中的所有 PDF，输出到 `D:\课程课件\输出笔记`。
```

## 使用方式二：在 ChatGPT 网页端使用

如果不使用 Codex，也可以直接使用 `prompt.md`。

步骤：

1. 在 ChatGPT 网页端上传一个或多个课程 PDF；
2. 复制 `prompt.md` 中的完整提示词；
3. 粘贴并发送；
4. 等待生成 Markdown 课程笔记。

适合临时处理少量课件。

## 使用方式三：作为个人笔记工作流规范

你也可以不依赖固定工具，只将本项目作为“课件转笔记”的格式规范。

适合用于：

* 个人 Obsidian 知识库；
* 团队培训资料整理；
* 课程复习资料标准化；
* AI 笔记处理流程设计。

## 输出格式

每个 PDF 应生成一份独立 Markdown 笔记。

如果输入：

```text
变电设备检修.pdf
继电保护基础.pdf
电力调度自动化.pdf
```

则输出：

```text
变电设备检修.md
继电保护基础.md
电力调度自动化.md
```

每份笔记一般包括：

```markdown
# 一级课程模块

## 主要知识点

### 局部内容分组

**加粗小标题：**
正文说明。

# 学习自测

## 一、单选题

## 二、多选题

## 三、判断题

## 四、简答题
```

注意：课程名通常由文件名体现，不建议再作为正文一级标题。

## 标题规则

本项目采用以下标题规则：

* `#`：课程中的一级模块；
* `##`：一级模块下的重要知识点；
* `###`：局部内容分组；
* 禁止使用 `####` 及以下标题；
* 不适合升级为正式标题的内容，使用加粗小标题。

这样可以避免 Markdown 层级过深，也便于 Obsidian 链接和长期维护。

## 内容组织规则

### 段落

适合用于：

* 概念解释；
* 原理说明；
* 背景意义；
* 连续逻辑；
* 理论性内容。

### 列表

适合用于：

* 多个并列知识点；
* 多个设备；
* 多个工程；
* 多个案例；
* 多个风险点；
* 多个控制措施；
* 多个适用场景；
* 多个优缺点。

### 表格

适合用于：

* 分类对比；
* 技术路线对比；
* 工程项目对比；
* 设备项目对比；
* 风险点与控制措施；
* 流程阶段；
* 检查项目与判断标准。

### 加粗小标题

适合用于：

* 典型工程；
* 典型案例；
* 项目示例；
* 单个设备说明；
* 局部技术特点；
* 某一操作要求；
* 某一注意事项。

## 学习自测规则

学习自测位于每份笔记末尾。

要求：

* 覆盖正文主要知识点；
* 不加入课件之外的新知识；
* 每道题包含答案和解析；
* 每道题关联正文原文标题；
* 链接格式适配 Obsidian。

示例格式：

```markdown
1. 关于某一知识点的说法，正确的是哪一项？  
   A. 选项  
   B. 选项  
   C. 选项  
   D. 选项  
   **答案：B**  
   **解析：** 解析内容。  
   关联原文：[[文件名#对应标题]]
```

## 设计原则

### 1. 内容完整优先

不为了简洁而删减知识点。

### 2. 结构服务学习

标题、段落、列表和表格都服务于学习与复习，而不是追求形式统一。

### 3. 避免 AI 味表达

避免使用过度口语化、网感化、模板化表达，例如：

* 一句话理解；
* 简单来说；
* 本质上就是；
* 重点来了；
* 超级重要；
* 秒懂；
* 保姆级；
* 小白也能懂。

### 4. 保留课程原意

允许整理表达、合并重复、优化结构，但不随意扩写课件外知识。

### 5. 适配长期维护

Markdown 文件应便于在 Obsidian 中继续修改、链接、复习和归档。

## 局限性

本项目依赖 AI 对 PDF 内容的提取和理解能力，因此可能存在以下问题：

* 扫描版 PDF 识别不完整；
* 图片、图表、流程图可能无法完全转写；
* 复杂表格可能需要人工校对；
* 部分课件中的隐含结构可能识别不准确；
* 学习自测题的链接需要检查是否指向真实标题。

因此，建议对生成结果进行人工复核，尤其是涉及参数、工程名称、安全要求、流程步骤和图表内容的部分。

## 推荐工作流

1. 准备课程 PDF；
2. 使用本 Skill 或 Prompt 生成 Markdown 笔记；
3. 检查标题结构；
4. 检查图表和参数；
5. 检查学习自测链接；
6. 将 Markdown 导入 Obsidian；
7. 根据个人学习习惯继续补充批注、标签和双链。

## 维护建议

如果发现生成结果存在重复问题，可以更新 `SKILL.md` 或 `prompt.md`。

适合固化进规则的问题：

* 标题层级经常错误；
* 学习自测链接经常失效；
* 参数密集内容可读性差；
* 工程案例没有独立呈现；
* 过度段落化或过度列表化；
* 某类课件经常遗漏图表说明。

不建议固化进规则的问题：

* 某一次课件的临时要求；
* 某一门课程特有的格式偏好；
* 一次性的题量调整；
* 临时要求不生成学习自测。
